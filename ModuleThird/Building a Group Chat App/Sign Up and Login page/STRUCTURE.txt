MERN Authentication App - Project Structure
==============================================

This is a complete MERN Stack application with separate backend and frontend folders.

==============

│   ├── routes/
│   │   └── auth.js            # Signup & Login endpoints
│   ├── server.js              # Express app & MongoDB connection
│   ├── package.json           # Backend dependencies
│   ├── .env                   # Database & JWT config
│   └── .gitignore
│
├── frontend/                   # React + Vite
│   ├── src/
│   │   ├── pages/
│   │   │   ├── Signup.jsx     # Registration form
│   │   │   ├── Login.jsx      # Login form
│   │   │   └── Dashboard.jsx  # User dashboard (protected)
│   │   ├── App.jsx            # Main app with routing
│   │   ├── api.js             # API calls to backend
│   │   ├── PrivateRoute.jsx   # Route protection

HOW TO RUN:
===========

1. BACKEND (Terminal 1):
   cd backend
   npm install
   npm start
   → Server runs on http://localhost:5000

2. FRONTEND (Terminal 2):
   cd frontend
   npm install
   npm run dev
   → App runs on http://localhost:3000

3. MongoDB:
   → Must be running (local or MongoDB Atlas)
   → Connection string in: backend/.env


KEY FILES & THEIR PURPOSE:
==========================

BACKEND:
--------
server.js          → Starts Express server, connects to MongoDB
auth.js            → API routes for /signup and /login
User.js            → MongoDB user model with bcrypt hashing
.env               → MongoDB URI and JWT secret

FRONTEND:
---------
App.jsx            → Main component with routing logic
api.js             → Functions to call backend API
Signup.jsx         → Registration page (name, email, phone, password)
Login.jsx          → Login page (email/phone + password)
Dashboard.jsx      → Protected page after login
PrivateRoute.jsx   → Protects routes from unauthenticated users


FEATURES:
=========

✓ Sign up with name, email, phone, password
✓ Login with email OR phone number
✓ Password hashing with bcryptjs
✓ JWT token authentication
✓ Protected dashboard (only logged-in users)
✓ Form validation (client & server)
✓ Modern UI with gradient styling
✓ Error handling & loading states
✓ Logout functionality
✓ Local storage for tokens


WHAT YOU CAN DO:
================

1. Sign up → Enter name, email, phone (10 digits), password (6+ chars)
2. Login → Use email or phone + password
3. View dashboard → See user info after login
4. Logout → Exit and return to login


ENVIRONMENT VARIABLES:
=====================

backend/.env:
  MONGODB_URI=mongodb://localhost:27017/auth_db
  JWT_SECRET=your_jwt_secret_key_change_this_in_production
  PORT=5000
  NODE_ENV=development


API ENDPOINTS:
==============

POST /api/auth/signup
  Body: { name, email, phone, password }
  Returns: { success, token, user }

POST /api/auth/login
  Body: { identifier (email/phone), password }
  Returns: { success, token, user }


TECHNOLOGIES USED:
==================

Backend:
  - Node.js + Express.js
  - MongoDB + Mongoose
  - bcryptjs (password hashing)
  - JWT (authentication)
  - express-validator (validation)
  - CORS enabled

Frontend:
  - React 18.2
  - Vite (build tool)
  - React Router v6 (routing)
  - Axios (API calls)
  - CSS3 (styling)


TROUBLESHOOTING:
================

Problem: "Cannot connect to MongoDB"
Solution: Ensure MongoDB is running or update MONGODB_URI in backend/.env

Problem: "Port 5000 already in use"
Solution: Kill process on port 5000 or change PORT in backend/.env

Problem: "CORS error"
Solution: Ensure both frontend and backend are running

Problem: "Token error on login"
Solution: Clear localStorage (F12 → Application → Clear All)


PRODUCTION CHECKLIST:
====================

□ Change JWT_SECRET in backend/.env to strong random string
□ Use MongoDB Atlas instead of local MongoDB
□ Deploy backend (Heroku, Railway, Render, etc)
□ Build frontend: cd frontend && npm run build
□ Deploy frontend (Vercel, Netlify, etc)
□ Update API URL in frontend/src/api.js to production backend
□ Enable HTTPS on both frontend and backend
□ Update CORS settings for production domain
